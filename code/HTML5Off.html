HTML5离线存储原理

web app中的一些资源并不经常改变，不需要每次都向服务器发送请求。这时应运而生的离线缓存就显得尤为突出

    步骤:
    1.只要在html头部加一个manifest属性就ok了
        < html manifest = "cache.manifest">
        < /html>
    2.然后在项目文件中新建一个后缀为cache.manifest文件,并书写方式如下：
        CACHE MANIFEST
        #v0.11   //这里写注释
        CACHE:   //要缓存的文件.这里写要缓存的文件路径  注：*号表示全部
        js/app.js
        css/style.css
        NETWORK:  //该属性表示不需要进行缓存,只能联网后才能访问;
        resourse/logo.png
        FALLBACK:  //该属性表示备份文件,如果上面缓存读取失败,会从这里面进行读取
        //404.html

            注意：如果服务器文件进行修改,那么需要把.manifest文件进行更新
            方法：手动更新,随便改一下.manifest文件里面的任意东西,用户访问的时候会及时更新
